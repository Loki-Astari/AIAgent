1. INTRODUCTION                                         *aiagent-introduction*

AIAgent.nvim integrates AI agent CLIs (Claude Code, Aider, Gemini, etc.) into
Neovim as a persistent terminal split. Features include:

  - Right-side terminal split with optional keybind header
  - Multiple named agents running simultaneously
  - Per-agent color theming via the agent's own /color command
  - Git worktree support (isolated branches per agent)
  - Visual selection and open-buffer context sending
  - LSP diagnostics integration (whole file or visual range)
  - Scroll mode with position memory
  - Bufferline tab integration
  - Idle attention alerts when a background agent finishes or needs input
  - Lualine integration: agent mode, worktree branch, and MCP server status

==============================================================================
2. REQUIREMENTS                                         *aiagent-requirements*

  - Neovim >= 0.9
  - At least one supported AI agent CLI installed in PATH

Supported CLIs:
  claude       Anthropic Claude Code
  cursor-agent Cursor AI
  aider        Aider (aider-chat)
  gemini       Google Gemini CLI
  codex        OpenAI Codex CLI
  goose        Block's Goose
  plandex      Plandex
  cody         Sourcegraph Cody
  amp          Amp

Optional:
  plenary.nvim Required only to run the test suite

3. SETUP                                                    *aiagent-setup*

Minimal setup (uses all defaults): >lua
  require('aiagent').setup()
<
Full example: >lua
  require('aiagent').setup({
    width               = 0.4,
    default_agent       = "claude",
    auto_send_context   = false,
    agent_startup_delay = 1500,
    show_header         = true,
    scroll_start_line   = 9,
    idle_timeout_ms     = 8000,
    idle_notify         = false,
    mcp_max_width       = 35,
    mcp_scroll          = true,
    colors = { "red", "blue", "orange", "green", "yellow",
               "magenta", "cyan", "purple" },
    known_agents = {
      claude  = "claude",
      aider   = "aider",
      my_tool = "my-cli-executable",  -- custom agent
    },
  })
<

4. CONFIGURATION                                           *aiagent-config*

All options are passed to |aiagent-setup|. Defaults are shown.

                                                     *aiagent-config-width*
width ~
  Type: number  Default: `0.4`
  Width of the terminal split.
  - 0 < width <= 1  →  fraction of total editor width
  - width > 1       →  absolute column count

                                              *aiagent-config-default_agent*
default_agent ~
  Type: string  Default: `"claude"`
  Symbolic name of the agent used for new terminals. Must be a key in
  |aiagent-config-known_agents|.

                                          *aiagent-config-auto_send_context*
auto_send_context ~
  Type: boolean  Default: `false`
  When `true`, automatically send unsent open buffers as @file context
  each time the agent window is focused.

                                        *aiagent-config-agent_startup_delay*
agent_startup_delay ~
  Type: number  Default: `1500`
  Milliseconds to wait after starting the agent before sending the
  `/color` command. Increase if your agent takes longer to initialize.

                                              *aiagent-config-show_header*
show_header ~
  Type: boolean  Default: `true`
  Show a non-interactive header above the terminal listing available
  keybindings.

                                          *aiagent-config-scroll_start_line*
scroll_start_line ~
  Type: number  Default: `9`
  Buffer line to jump to when entering scroll mode for the first time
  in a session. Skips the agent's startup preamble.

                                          *aiagent-config-idle_timeout_ms*
idle_timeout_ms ~
  Type: number  Default: `8000`
  Milliseconds of silence after activity before a background agent is
  flagged as needing attention. Set to `0` to disable idle alerts
  entirely. See |aiagent-idle|.

                                              *aiagent-config-idle_notify*
idle_notify ~
  Type: boolean  Default: `false`
  When `true`, also fire |vim.notify| when a background agent is flagged.
  The tab indicator is always shown regardless of this setting.
  See |aiagent-idle|.

                                                 *aiagent-config-colors*
colors ~
  Type: string[]
  Default: `{ "red", "blue", "orange", "green", "yellow", "magenta", "cyan", "purple" }`
  Color cycle used when opening new agents. Each new agent gets the next
  color. Colors are sent to the agent via its `/color` command.

                                           *aiagent-config-mcp_max_width*
mcp_max_width ~
  Type: number  Default: `35`
  Maximum display-column width for the MCP status component in lualine
  before the carousel scroll activates. Increase this if your server
  names are long and you want more of them visible at once.
  See |aiagent-lualine-mcp|.

                                              *aiagent-config-mcp_scroll*
mcp_scroll ~
  Type: boolean  Default: `true`
  When `true`, the MCP status component scrolls through server names in
  a carousel when the total width exceeds |aiagent-config-mcp_max_width|.
  Set to `false` to always show the full list without scrolling.
  See |aiagent-lualine-mcp|.

                                            *aiagent-config-known_agents*
known_agents ~
  Type: table<string, string>  (symbolic name → CLI executable)
  Map of symbolic agent names to CLI executables. Extend this to add
  custom agents: >lua
    known_agents = { my_agent = "my-cli-binary" }
<

5. COMMANDS                                               *aiagent-commands*

                                                              *:AgentOpen*
:AgentOpen [Name [WTName [directory]]]
  Open an agent terminal. All arguments are optional.

  Name        Agent name (default: "AIAgent"). If an agent with this
              name is already running, switches to it instead.
  WTName      Worktree name. "-" is shorthand for using Name as the
              worktree name. See |aiagent-worktrees|.
  directory   Explicit directory for a new worktree. Error if the
              worktree already exists.

                                                             *:AgentClose*
:AgentClose [Name]
  Close the named agent (default: current agent). If it was the last
  agent, the terminal window is also closed.

                                                            *:AgentToggle*
:AgentToggle [Name]
  Toggle the agent window. Opens if closed, closes if the named agent
  is currently visible.

                                                            *:AgentSwitch*
:AgentSwitch {Name}
  Switch the terminal window to display the named agent.

                                                               *:AgentSet*
:AgentSet {type}
  Set the agent type used for subsequent |:AgentOpen| calls.
  {type} must be a key in |aiagent-config-known_agents|.

                                                          *:AgentSetColor*
:AgentSetColor {color}
  Change the color of the current agent. Sends `/color {color}` to the
  CLI and refreshes the winbar tab.

                                                              *:AgentList*
:AgentList
  Print a list of running agents with their types. The current agent
  is marked with `*`.

                                                          *:AgentCloseAll*
:AgentCloseAll
  Close all running agents and the terminal window.

                                                       *:AgentSendContext*
:AgentSendContext
  Send all open buffer file paths to the current agent as @file
  references. Already-sent files are skipped.

                                                      *:AgentResetContext*
:AgentResetContext
  Clear the list of files that have been sent as context. The next
  |:AgentSendContext| will re-send all open buffers.

                                                     *:AgentSendSelection*
:{range}AgentSendSelection
  Send the visual selection to the current agent as a markdown code
  block. Opens the agent if not already running.

                                                  *:AgentSendDiagnostics*
:{range}AgentSendDiagnostics
  Collect LSP diagnostics for the current buffer and send them to the
  agent as a formatted prompt. The agent is focused and left in insert
  mode with the text ready — press Enter to start the analysis.

  Without a range, all diagnostics in the current file are included.
  With a range (e.g. `:'<,'>AgentSendDiagnostics`), only diagnostics
  whose line falls within the selected lines are included.

  The prompt includes:
    - A brief description asking the agent to analyse and suggest fixes
    - The full file path
    - Active language server name(s) and their compiler/workspace settings
    - All matching diagnostics sorted by line and column, wrapped in a
      fenced ```` ```<Errors> ```` block with severity, line, column,
      message, and source

  Diagnostics are sorted by line then column. Severity labels used:
    ERROR   vim.diagnostic.severity.ERROR
    WARN    vim.diagnostic.severity.WARN
    INFO    vim.diagnostic.severity.INFO
    HINT    vim.diagnostic.severity.HINT

  Opens the agent if not already running.
  See also |aiagent.send_diagnostics()|.

6. KEYMAPS                                                 *aiagent-keymaps*

These keymaps are active inside the agent terminal buffer.

Terminal mode ~
  <C-\><C-n>    Exit terminal mode and return to the previous window
  <C-\><C-s>    Enter scroll mode (stay in agent window)
  <C-\><C-a>    Cycle to the next agent
  <C-\><C-c>    Send open buffer context (see |:AgentSendContext|)
  <C-\><C-v>    Paste the unnamed register `"` into the terminal input
  <C-\><C-r>    Search input history (agent-specific)

Scroll mode (normal mode inside terminal buffer) ~
  i             Exit scroll mode and resume terminal input
  <C-\><C-n>    Exit scroll mode and return to the previous window
  Any normal-mode motion (j, k, /, ?, G, etc.) to browse history

Scroll mode position memory ~
  - First entry: jumps to |aiagent-config-scroll_start_line|
  - Re-entry: restores the last cursor position from the previous
    scroll session

7. LUA API                                                    *aiagent-api*

All public functions are on the `require('aiagent')` table.

                                                    *aiagent.setup()*
setup({opts})
  Initialize the plugin and register autocmds. Safe to call multiple
  times (autocmds are cleared and re-registered via the AIAgent augroup).
  See |aiagent-config| for available options.

                                                     *aiagent.open()*
open({name}, {wtname}, {directory})
  Programmatic equivalent of |:AgentOpen|.

                                                    *aiagent.close()*
close({name})
  Programmatic equivalent of |:AgentClose|.

                                                   *aiagent.toggle()*
toggle({name})
  Programmatic equivalent of |:AgentToggle|.

                                                   *aiagent.switch()*
switch({name})
  Programmatic equivalent of |:AgentSwitch|.

                                                      *aiagent.set()*
set({type})
  Programmatic equivalent of |:AgentSet|.

                                                *aiagent.set_color()*
set_color({color})
  Programmatic equivalent of |:AgentSetColor|.

                                             *aiagent.send_context()*
send_context({agent_name})
  Send unsent open buffers as context. Returns the number of files sent.

                                            *aiagent.reset_context()*
reset_context({agent_name})
  Clear sent-files tracking so all buffers will be re-sent next time.

                                         *aiagent.pending_context_count()*
pending_context_count({agent_name})
  Return the number of open buffers not yet sent as context.

                                          *aiagent.send_selection()*
send_selection({agent_name})
  Send the last visual selection as a markdown code block.

                                        *aiagent.send_diagnostics()*
send_diagnostics({agent_name}, {line1}, {line2})
  Collect LSP diagnostics for the current buffer and send a formatted
  prompt to the agent. Programmatic equivalent of |:AgentSendDiagnostics|.

  {agent_name}  string|nil  Agent to target (default: current agent)
  {line1}       number|nil  First line of range, 1-indexed (nil = whole file)
  {line2}       number|nil  Last line of range, 1-indexed (nil = whole file)

  Before sending, `\x1bi` is written to the terminal pty so that agents
  using vim keybindings (e.g. Claude's vim mode) are guaranteed to be in
  insert mode.

                                                 *aiagent.is_open()*
is_open()
  Return `true` if the terminal window is currently visible.

                                                    *aiagent.list()*
list()
  Return a list of running agent names (sorted alphabetically).

                                              *aiagent.print_list()*
print_list()
  Notify the running agents via |vim.notify|. Equivalent of |:AgentList|.

                                              *aiagent.close_all()*
close_all()
  Close all agents and windows. Equivalent of |:AgentCloseAll|.

                                   *aiagent.bufferline_name_formatter()*
bufferline_name_formatter({buf})
  Name formatter for bufferline.nvim. See |aiagent-bufferline|.

                                                *aiagent.lualine_label()*
lualine_label()
  Returns a display string for lualine section A when the current buffer
  is the active agent terminal. Returns `nil` otherwise (caller should
  fall back to the normal mode string).

  Values returned:
    `"Agent: Claude"`       agent terminal in input mode
    `"Scroll Mode: Claude"` agent terminal in scroll mode
    nil                     any other buffer

                                                *aiagent.lualine_color()*
lualine_color()
  Returns a lualine color table for the section A component, or `nil`
  when not in the agent terminal (allowing the theme default).

  Values returned:
    `{ bg = '#0891b2', fg = '#ffffff' }`  input mode  (cyan)
    `{ bg = '#7c3aed', fg = '#ffffff' }`  scroll mode (purple)
    nil                                   other buffers

                                               *aiagent.lualine_branch()*
lualine_branch()
  Returns the current git branch of the active agent's worktree when
  the agent terminal is focused (determined via `git branch --show-current`
  in the worktree directory). Returns `nil` otherwise.

  Typical lualine usage (falls back to gitsigns for normal buffers): >lua
    function()
      return require('aiagent').lualine_branch()
          or vim.b.gitsigns_head or ''
    end
<
                                                  *aiagent.lualine_mcp()*
lualine_mcp()
  Returns a string listing all connected MCP servers, suitable for a
  lualine component. See |aiagent-lualine-mcp|.

                                            *aiagent.lualine_mcp_color()*
lualine_mcp_color()
  Returns `{ fg = '#22c55e' }` (green) once the MCP server list has
  loaded, or `{ fg = '#94a3b8' }` (grey) while the initial query is
  still running.

                                                 *aiagent.mcp_refresh()*
mcp_refresh()
  Clears the MCP server cache and immediately triggers a fresh
  `claude mcp list` query. Use this after changing your Claude MCP
  configuration so the statusline updates without waiting for the
  30-second cache to expire.

8. GIT WORKTREES                                          *aiagent-worktrees*

Each agent can be associated with a git worktree — an isolated checkout of
the repository on a dedicated branch. Worktrees are persistent across Neovim
sessions.

Naming convention ~
  Branch:    `agent/{slug}`
  Directory: `$TMPDIR/nvim-agent-{repo}-{slug}`

  Where {slug} is the worktree name lowercased with non-alphanumeric
  characters replaced by `-`, and {repo} is the repository directory name
  (prevents collisions across different repos).

Opening with a worktree ~
>
  :AgentOpen MyFeature -          " uses "MyFeature" as WTName
  :AgentOpen MyFeature MyFeature  " same, explicit
  :AgentOpen MyFeature feat /path " creates worktree at /path
<

Auto-reconnect ~
  On `:AgentOpen Name` (no WTName), the plugin scans the git worktree
  list for a worktree whose branch matches `refs/heads/agent/{slug}`.
  If found, the agent reconnects to it silently.

File redirection ~
  While a worktree agent is active, opening files with `:e` inside the
  same repository automatically redirects to the worktree version.
  Bufferline tabs for redirected files are prefixed with the worktree
  slug (requires |aiagent-bufferline| configuration).

  To bypass redirection: >
    :noautocmd e path/to/file
<
Removing a worktree ~
>
  git worktree remove /path/to/worktree
  git branch -d agent/my-feature
<

9. IDLE ATTENTION ALERTS                                      *aiagent-idle*

When you start a long-running job in one agent and switch to another, the
plugin watches for the first agent to finish or pause for input. When a
background agent has produced new output and then gone silent for
|aiagent-config-idle_timeout_ms| milliseconds, its winbar tab is highlighted
and a `●` symbol is appended to the name: >

  [ Review ] [ Feature ● ]
<
The alert clears automatically when you switch back to that agent. If the
agent resumes work (produces more output), the alert is also cleared
immediately.

Detection ~
  The plugin records the terminal buffer's line count each time you visit
  an agent. While the agent is in the background:

  - New lines added to the buffer update an internal timestamp
  - Cursor blinks, prompt redraws, and in-place updates (which do not
    add lines) are ignored — they cannot re-trigger an alert after a visit
  - Once the new-line timestamp is older than |aiagent-config-idle_timeout_ms|,
    the tab is flagged

  Agents that are idle because they have not been asked anything are never
  flagged — only agents that were actively producing output and then stopped.

Configuration ~
  See |aiagent-config-idle_timeout_ms| and |aiagent-config-idle_notify|.

  To disable idle alerts: >lua
    require('aiagent').setup({ idle_timeout_ms = 0 })
<
  To also receive a |vim.notify| message when an agent is flagged: >lua
    require('aiagent').setup({ idle_notify = true })
<

10. LSP DIAGNOSTICS                                     *aiagent-diagnostics*

|:AgentSendDiagnostics| collects LSP diagnostics for the current buffer via
`vim.diagnostic.get()` and sends a structured prompt to the agent.

Whole-file usage ~
>
  :AgentSendDiagnostics
<
  All diagnostics for the current buffer are included.

Range usage ~
>
  :'<,'>AgentSendDiagnostics
<
  Only diagnostics whose line falls within the visual selection are sent.
  Useful for focusing on a specific function or block.

Prompt structure ~

  The message typed into the agent prompt contains:

    1. A brief instruction asking the agent to analyse and suggest fixes
    2. `**File:** /absolute/path/to/file`
    3. `**Language Server(s) and compiler options:**` listing each active
       LSP client and its `config.settings` (or `init_options`) as JSON
    4. `**Errors:**` followed by a fenced block:
>
       ```<Errors>
       [ERROR] line 10, col 5: Undefined variable 'x' (lua_ls)
       [WARN]  line 14, col 1: Unused import (lua_ls)
       ```
<
    5. A closing instruction to explain what is wrong and how to fix it

  The text is NOT submitted automatically — the agent is focused in insert
  mode with the prompt ready, and the user presses Enter to send.

Vim mode compatibility ~

  Before the text is sent, `<Esc>i` is written to the terminal pty. This
  ensures agents with vim keybindings (such as Claude's vim mode) are in
  insert mode regardless of their previous state.

Suggested mapping ~
>lua
  vim.keymap.set("n", "<leader>ad", "<cmd>AgentSendDiagnostics<cr>",
    { desc = "Send LSP diagnostics to agent" })
  vim.keymap.set("v", "<leader>ad", "<cmd>AgentSendDiagnostics<cr>",
    { desc = "Send LSP diagnostics (selection) to agent" })
<

11. BUFFERLINE INTEGRATION                              *aiagent-bufferline*

To prefix worktree-redirected buffer tabs with the worktree slug, pass the
`bufferline_name_formatter` function to bufferline's `name_formatter` option:
>lua
  require('bufferline').setup({
    options = {
      name_formatter = require('aiagent').bufferline_name_formatter,
    },
  })
<
Tabs for worktree files will display as `slug: filename` (e.g.
`feature: main.cpp`). Non-worktree buffers are unaffected.

12. LUALINE INTEGRATION                                    *aiagent-lualine*

AIAgent provides optional helpers for lualine.nvim that surface agent state
in your statusline. None of these require any setup beyond adding them to
your lualine configuration.

Section A — Agent mode ~                              *aiagent-lualine-mode*

  Replaces the standard mode indicator with the agent name and current
  mode when the agent terminal window is focused: >lua

    lualine_a = {
      {
        function()
          return require('aiagent').lualine_label()
              or require('lualine.utils.mode').get_mode()
        end,
        color = function()
          return require('aiagent').lualine_color()
        end,
      },
    },
<
  The component color changes automatically:
    Cyan   (#0891b2)  when in normal agent input mode
    Purple (#7c3aed)  when in scroll mode
    Theme default     for all other buffers

Section B — Worktree branch ~                        *aiagent-lualine-branch*

  Shows the real git branch of the active agent's worktree when the agent
  terminal is focused. Falls back to gitsigns for all other buffers: >lua

    lualine_b = {
      {
        function()
          return require('aiagent').lualine_branch()
              or vim.b.gitsigns_head or ''
        end,
        icon = '\u{E0A0}',
      },
      'diff', 'diagnostics',
    },
<
  The branch is read with `git -C {worktree} branch --show-current` so it
  reflects the actual checked-out branch, not an assumed name.

Section X — MCP server status ~                         *aiagent-lualine-mcp*

  Displays connected MCP servers queried from `claude mcp list`. Results
  are cached for 30 seconds. Only servers reported as `✓ Connected` are
  shown; auto-discovered `claude.ai` servers are filtered out. >lua

    lualine_x = {
      {
        function() return require('aiagent').lualine_mcp() end,
        color = function() return require('aiagent').lualine_mcp_color() end,
      },
      'encoding', 'filetype',
    },
<
  Example output: >
    MCP: ✓ atlassian  ✓ github
<

Carousel scroll ~                               *aiagent-lualine-mcp-scroll*

  When the MCP server list is wider than |aiagent-config-mcp_max_width|
  columns, the component switches to a fixed-width scrolling carousel:

  - A window of `mcp_max_width` columns slides through the server list
  - The window advances one character every 300 ms via a `vim.uv` timer
  - The loop is seamless: a separator gap (`   `) is appended before the
    text repeats, so the scroll wraps smoothly
  - The component is padded to a fixed width so the statusline does not
    resize as the text scrolls
  - The timer is stopped automatically when the content fits within
    `mcp_max_width` (e.g. after servers disconnect) or when there are no
    servers to display

  To disable scrolling: >lua
    require('aiagent').setup({ mcp_scroll = false })
<
  To widen the display area: >lua
    require('aiagent').setup({ mcp_max_width = 50 })
<
  To force an immediate refresh after changing Claude config: >lua
    require('aiagent').mcp_refresh()
<
